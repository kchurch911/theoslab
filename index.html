🧭 자기주도학습(SDL)이란?
Malcolm Knowles(1975)의 SDL 이론에 따르면, 자기주도학습자는 다음을 스스로 수행합니다:

(1) 학습 필요성 진단 → (2) 목표 설정 → (3) 자원 확보 → (4) 학습 전략 선택 → (5) 평가
TheosLab SDL 7원리
01
자율성 (Autonomy)
내가 배우는 이유를 외부가 아닌 내면에서 찾는다. "하나님 앞에서 내가 왜 이것을 배우는가?" 매일 목적을 확인한다.
02
반성적 저널링 (Reflective Journaling)
매일 학습 후 3가지를 기록: ① 오늘 배운 것, ② 연결된 사역 적용, ③ 내일 탐구할 질문
03
간격 반복 (Spaced Repetition)
새 개념은 1일→3일→7일→30일 간격으로 복습. Anki 또는 직접 만든 복습 카드 활용.
04
페인만 기법 (Feynman Technique)
배운 개념을 교인/자녀에게 설명하는 연습. "설명할 수 없으면 아직 모르는 것이다."
05
즉각 적용 (Immediate Application)
배운 것을 72시간 이내에 실제 설교, 코드, 논문에 적용. 이론과 실천의 격차를 최소화.
06
통합적 연결 (Cross-Track Integration)
매주 토요일: 이번 주 배운 5트랙을 하나의 사역 적용으로 연결하는 통합 노트 작성.
07
공개 설명 (Teaching as Learning)
매월 주일설교나 목회칼럼에서 TheosLab 학습 내용 1가지 적용·나눔. 가르침이 최고의 학습이다.

① 플랫폼 독립성

③ JVM 위의 다른 언어들
Kotlin·Scala·Groovy도 JVM에서 실행 → Android(Kotlin)가 Java와 완벽 호환.

📚 코딩 핵심 용어 사전 (가나다 순)
API (Application Programming Interface)
프론트엔드와 백엔드가 약속된 방식으로 데이터를 주고받는 인터페이스. REST API가 가장 일반적.
CDN (Content Delivery Network)
전 세계 곳곳에 서버를 배치해 사용자 가까운 곳에서 파일 전송. 속도 향상. Cloudflare·AWS CloudFront.
Docker (컨테이너)
앱과 실행 환경을 하나의 박스(컨테이너)에 담아 어디서든 동일하게 실행. "내 PC에서는 되는데…" 해결.
Firebase
Google의 BaaS(Backend as a Service). DB·인증·호스팅·스토리지를 코드 없이 사용. Hebron 앱 핵심.
Framework (프레임워크)
개발의 틀. 공통 기능을 미리 만들어 놓고 개발자가 채워 넣음. React(JS), Spring(Java), Django(Python).
Git / GitHub
코드 버전 관리 도구. 변경 이력 추적·협업·백업. GitHub는 클라우드 저장소. SeattleGuide 호스팅 중.
HTTP / HTTPS
웹 통신 프로토콜(약속). HTTPS는 암호화 추가. 브라우저↔서버의 대화 방식. 상태코드(200·404·500).
JSON (JavaScript Object Notation)
데이터 교환 형식. {key:value} 구조. API 응답 표준. {"name":"폴","role":"pastor"} 형태.
JVM (Java Virtual Machine)
Java 바이트코드를 실행하는 가상 머신. OS에 관계없이 실행 가능(WORA). Kotlin·Scala도 JVM 위에서 동작.
Library (라이브러리)
특정 기능 모음. 개발자가 필요할 때 꺼내 씀. 프레임워크와 달리 흐름을 개발자가 제어.
MCP (Model Context Protocol)
Claude 등 AI 모델이 외부 도구·데이터에 접근하는 표준 프로토콜. 파일·DB·API 연결. T6 핵심.
PWA (Progressive Web App)
웹앱을 마치 네이티브 앱처럼. 홈 화면 추가·오프라인 동작·푸시 알림. SeattleGuide·서북미수련회앱.
REST API
HTTP 기반의 API 설계 원칙. GET/POST/PUT/DELETE 메서드로 자원(Resource) 조작. URL로 자원 표현.
SaaS (Software as a Service)
소프트웨어를 구독 방식으로 제공. 설치 불필요. TheosLab·HebronApp의 수익 모델 목표.
WAS (Web Application Server)
동적 요청 처리 서버. 비즈니스 로직·DB 연동·세션 관리. Tomcat·Spring Boot·Node.js·Django.
웹 서버 (Web Server)
정적 파일(HTML·CSS·이미지) 제공 전문 서버. Nginx·Apache. 로드밸런싱·SSL 처리도 담당.
